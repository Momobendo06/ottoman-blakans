Les données brutes extraites de Wikidata contiennent des doublons, des labels non résolus et des batailles hors périmètre. Ce script Python les transforme en un corpus propre et analysable.

Vue d'ensemble

Le script en un coup d'œil

Le script clean_ottoman_battles.py prend en entrée le CSV brut exporté par l'extracteur Wikidata et produit un CSV nettoyé prêt pour l'analyse. Il s'utilise en ligne de commande :

Utilisation BASH

python clean_ottoman_battles.py --input mon_fichier.csv

01
Étape 1

Résolution des labels non résolus

Wikidata retourne parfois un identifiant brut (QID) au lieu d'un libellé lisible, lorsque l'objet n'a pas de nom défini en anglais ou en français. Un dictionnaire de corrections manuelles (LABEL_FIXES) remplace ces QID par leurs noms corrects.

clean_ottoman_battles.py — étape 1 PYTHON 
LABEL_FIXES = {
    "Q115732142": "Battle of Sântimbru (March 27)",
    "Q136033321": "Battle of Schladming",
    "Q9333973":   "Sasiv Horn",   # lieu en Ukraine, filtré ensuite
}

def resolve_labels(rows):
    for row in rows:
        for field in ("battleLabel", "locationLabel"):
            if row[field] in LABEL_FIXES:
                row[field] = LABEL_FIXES[row[field]]
    return rows
02
Étape 2

Filtre périmètre ottoman-balkanique

Le filtre géographique SPARQL (lat 35–48°N, lon 13–30°E) capture parfois des batailles sans lien avec les Ottomans : guerres vénitiennes, conflits aragonais, révoltes siciliennes. Une liste noire de guerres hors périmètre (OUT_OF_SCOPE_WARS) permet de les exclure, à condition qu'elles ne contiennent aucun mot-clé ottoman dans leurs champs.

clean_ottoman_battles.py — étape 2 PYTHON 
OUT_OF_SCOPE_WARS = {
    "First Anglo-Dutch War",
    "Franco-Dutch War",
    "Aragonese conquest of Naples",
    "War of Chioggia",
    "Venetian–Genoese Wars",
    ...
}

OTTOMAN_KEYWORDS = ["ottoman", "turk", "dardanelles", "gallipoli"]

def filter_out_of_scope(rows):
    kept = []
    for row in rows:
        in_blacklist = row["part_ofLabel"] in OUT_OF_SCOPE_WARS
        is_ottoman   = any(kw in (row["battleLabel"] + row["part_ofLabel"]).lower()
                           for kw in OTTOMAN_KEYWORDS)
        if not (in_blacklist and not is_ottoman):
            kept.append(row)
    return kept
03
Étape 3

Déduplication

Une même bataille peut apparaître plusieurs fois dans Wikidata si elle est associée à plusieurs lieux ou plusieurs conflits parents. Le script groupe les lignes par (wikidata_url, date) et :

— retient le lieu le plus spécifique (un château plutôt qu'une mer ou un fleuve)
— fusionne les conflits parents distincts séparés par un « | »

clean_ottoman_battles.py — étape 3 PYTHON 
def location_specificity_score(location):
    # Les termes génériques (mer, golfe, fleuve) ont un score élevé
    generic = ["sea", "ocean", "gulf", "river", "aegean", "adriatic"]
    return sum(1 for term in generic if term in location.lower())

def deduplicate(rows):
    groups = defaultdict(list)
    for row in rows:
        groups[(row["wikidata_url"], row["date"])].append(row)

    result = []
    for (url, date), group in groups.items():
        best = min(group, key=lambda r: location_specificity_score(r["locationLabel"]))
        all_parts = sorted({r["part_ofLabel"] for r in group if r["part_ofLabel"]})
        best["part_ofLabel"] = " | ".join(all_parts)
        result.append(best)
    return result

Étape 4

Résolution des conflits de dates

Certaines batailles ont deux dates dans Wikidata : une date approximative (1389-01-01, seule l'année est connue) et une date précise (1389-07-06). Le script conserve la date précise et supprime l'approximative. Si plusieurs dates précises distinctes existent, ce sont des événements complétement différents — ils sont tous conservés.

clean_ottoman_battles.py — étape 4 PYTHON 
def resolve_multi_dates(rows):
    groups = defaultdict(list)
    for row in rows:
        groups[row["wikidata_url"]].append(row)

    result = []
    for url, group in groups.items():
        precise = [r for r in group if date_precision(r["date"]) == "day"]
        approx  = [r for r in group if date_precision(r["date"]) != "day"]

        if precise:
            precise_dates = {r["date"] for r in precise}
            if len(precise_dates) == 1:
                result.append(precise[0])   # date précise unique → on garde
            else:
                result.extend(precise)      # dates distinctes → événements différents
        else:
            result.append(approx[0])        # que des approx → garder la première
    return result

Étape 5

Ajout de la colonne date_precision

Wikidata encode la précision des dates de manière implicite : YYYY-01-01 signifie "seule l'année est connue", YYYY-MM-01 signifie "seul le mois est connu". Ce script ajoute une colonne date_precision explicite (year / month / day) pour faciliter les analyses temporelles.

clean_ottoman_battles.py — étape 5 PYTHON 
def date_precision(date_str):
    parts = date_str.split("-")
    if parts[1] == "01" and parts[2] == "01":
        return "year"    # ex: 1389-01-01 → seule l'année est connue
    if parts[2] == "01":
        return "month"   # ex: 1389-07-01 → mois connu
    return "day"         # ex: 1389-07-06 → date complète

# Résultat dans le corpus nettoyé :
# day   : 89 batailles  (45%)
# year  : 87 batailles  (44%)
# month : 22 batailles  (11%)

Étape 6

Harmonisation des noms de guerres

Wikidata appelle parfois un conflit par différentes appellations pour désigner le même conflit. Un dictionnaire d'harmonisation (WAR_HARMONIZATION) normalise ces variantes vers une forme canonique unique, évitant ainsi que le même conflit apparaisse sous deux nœuds distincts dans l'analyse de réseau.

clean_ottoman_battles.py — étape 6 PYTHON 
WAR_HARMONIZATION = {
    "Byzantine–Ottoman wars":          "Byzantine–Ottoman Wars",
    "Hungarian–Ottoman War":           "Ottoman–Hungarian Wars",
    "Venetian–Ottoman Wars":           "Ottoman–Venetian Wars",
    "Cretan War of 1645-1669":         "Cretan War (1645–1669)",
    "Austro-Turkish War of 1663–1664": "Austro-Turkish War (1663–1664)",
    "Polish–Ottoman War of 1683–99":   "Great Turkish War (1683–1699)",
    "Great Turkish War":               "Great Turkish War (1683–1699)",
}

def harmonize_wars(rows):
    for row in rows:
        original = row.get("part_ofLabel", "")
        row["part_ofLabel"] = WAR_HARMONIZATION.get(original, original)
    return rows
